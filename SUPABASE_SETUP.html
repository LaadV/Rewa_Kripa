Step 2 of 6
3

Run the Database Setup SQL

This creates the tables needed for seat booking

1
In your Supabase project, click "SQL Editor" in the left sidebar (looks like </> icon)
2
Click "New query" button (top left of SQL editor)
3
Copy the entire SQL below and paste it into the query box
4
Click the "Run" button (green, bottom right of editor)
5
You should see a table with 2 rows: seats and site_config at the bottom
-- ═══════════════════════════════════════════════════════ -- REWA KRIPA TRAVELS — Supabase Setup (PASTE & RUN ALL) -- ═══════════════════════════════════════════════════════ -- 1. Seats table create table if not exists seats ( id bigint generated always as identity primary key, bus_id text not null, travel_date date not null, seat_num int not null, gender text not null default 'M', passenger_name text default '', passenger_phone text default '', status text not null default 'booked', booked_at timestamptz default now(), constraint seats_unique unique (bus_id, travel_date, seat_num), constraint gender_check check (gender in ('M','F')) ); -- 2. Site config table create table if not exists site_config ( id int primary key default 1, config_json jsonb, updated_at timestamptz default now() ); -- 3. Enable security alter table seats enable row level security; alter table site_config enable row level security; -- 4. Drop old policies (safe to re-run) drop policy if exists "seats_read" on seats; drop policy if exists "seats_write" on seats; drop policy if exists "seats_update" on seats; drop policy if exists "seats_delete" on seats; drop policy if exists "seats_select" on seats; drop policy if exists "seats_insert" on seats; drop policy if exists "seats_all" on seats; drop policy if exists "config_read" on site_config; drop policy if exists "config_write" on site_config; drop policy if exists "config_all" on site_config; drop policy if exists "config_select" on site_config; -- 5. Allow public access (booking page + admin) create policy "seats_select" on seats for select using (true); create policy "seats_insert" on seats for insert with check (true); create policy "seats_update" on seats for update using (true); create policy "seats_delete" on seats for delete using (true); create policy "config_select" on site_config for select using (true); create policy "config_all" on site_config for all using (true) with check (true); -- 6. Enable real-time do $$ begin begin alter publication supabase_realtime add table seats; exception when others then null; end; end $$; -- 7. Index for faster queries create index if not exists idx_seats_bus_date on seats (bus_id, travel_date); -- 8. Verify (should show 2 rows: seats + site_config) select table_name from information_schema.tables where table_schema = 'public' and table_name in ('seats', 'site_config') order by table_name;
✅
Expected result after running: A table at the bottom showing 2 rows — "seats" and "site_config". If you see this, the SQL worked!
⚠️
If you get an error saying "policy already exists" — that's fine, it means the DROP commands didn't catch everything. The tables were still created. Just click Continue below.
Common Error Messages & Fixes
Error MessageWhat To Do
policy "seats_select" already exists
 Ignore it — this just means you ran the SQL before. Tables are still created correctly.
relation "seats" already exists
 Table already exists — that's fine! The "IF NOT EXISTS" means it won't overwrite data.
permission denied for schema public
 You're logged in as read-only user. Make sure you're using SQL Editor inside your own project (not a shared project).
syntax error at or near "do"
 The realtime block failed — skip it. Remove lines 38-42 (the "do $$ begin..." block) and run again. The rest still works.
